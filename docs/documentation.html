Hybrid Framework Documentation

Getting Started

This documentation will guide you through using the Hybrid Framework to develop and manage your Android applications.

1. Changing App Information

Use this command to change your app's name and package name:

npm run change-app-info

This command will:

  • Ask for a new app name
  • Ask for a new package name
  • Update both package.json and package-lock.json files
Note: Package name should follow Java package naming conventions (e.g., com.example.myapp)

2. Managing Android Permissions

Use this command to manage Android permissions for your app:

npm run manage-permissions

You can select from these permissions:

  • INTERNET - For accessing the internet
  • CAMERA - For using the device's camera
  • READ_EXTERNAL_STORAGE - For reading files from storage
  • WRITE_EXTERNAL_STORAGE - For writing files to storage
  • ACCESS_FINE_LOCATION - For precise location access
  • ACCESS_COARSE_LOCATION - For approximate location access
  • And many more...

3. Building Your App

Use these commands to build your application:

npm run build
npm run build:android

The build process will:

  • Compile your code
  • Generate the Android APK
  • Place the APK in the appropriate directory

4. Updating Your App

To update your app:

  1. Make your changes to the source code
  2. Update the version number in package.json
  3. Run the build commands:
    4. npm run build
    npm run build:android
  4. Test the new APK
  5. Deploy the updated APK to your users

5. Important Notes

Version Management

When updating your app, always increment the version number in package.json. Follow semantic versioning (MAJOR.MINOR.PATCH).

Backup

Before making major changes, create a backup of your project files.

Testing

Always test your app thoroughly after making changes, especially after updating permissions or changing app information.

6. Troubleshooting

Common Issues and Solutions:

  • Permission Issues: If your app crashes after changing permissions, make sure to:
    • Request permissions at runtime if they are dangerous permissions
    • Handle permission denial gracefully in your code
  • Package Name Issues: If you change the package name:
    • Make sure to update all references to the old package name
    • Rebuild the app completely
  • Build Issues: If the build fails:
    • Check for syntax errors in your code
    • Make sure all dependencies are installed
    • Check the build logs for specific error messages
